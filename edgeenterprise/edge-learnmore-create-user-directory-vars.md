<span data-ttu-id="69096-107">Richtlinien zum Ändern von Datenverzeichnispfaden (zum Beispiel das Konfigurieren der Unterstützungsvariablen [UserDataDir](microsoft-edge-policies.md#userdatadir) oder [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory).</span><span class="sxs-lookup"><span data-stu-id="69096-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="69096-108">Beim Konfigurieren dieser Richtlinien können Sie Variablen anstelle hartcodierter Pfade verwenden.</span><span class="sxs-lookup"><span data-stu-id="69096-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="69096-109">Zum Beispiel, um Ihre Profildaten unter Windows unter lokalen Benutzeranwendungsdaten anstelle des Standardverzeichnisses zu speichern.</span><span class="sxs-lookup"><span data-stu-id="69096-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="69096-110">Legen Sie die Richtlinie [UserDataDir](microsoft-edge-policies.md#userdatadir) auf **${local_app_data}\Edge\Profile** fest.</span><span class="sxs-lookup"><span data-stu-id="69096-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="69096-111">Bei den meisten Windows10-Installationen wird dieser Pfad in *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile* aufgelöst.</span><span class="sxs-lookup"><span data-stu-id="69096-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

><span data-ttu-id="69096-112">Wenn nicht festgelegt, verwendet Microsoft Edge Standardverzeichnispfade.</span><span class="sxs-lookup"><span data-stu-id="69096-112">If left unset, Microsoft Edge uses default directory paths.</span></span> <span data-ttu-id="69096-113">Ein Benutzer kann das `--user-data-dir`-Befehlszeilenkennzeichen verwenden, um die Standardeinstellung außer Kraft zu setzen.</span><span class="sxs-lookup"><span data-stu-id="69096-113">A user can use the `--user-data-dir` command-line flag to override the default.</span></span>

### <span data-ttu-id="69096-116">Leitfaden für die Verwendung von Pfadvariablen</span><span class="sxs-lookup"><span data-stu-id="69096-116">Guidance for using path variables</span></span>

<span data-ttu-id="69096-117">Lesen Sie die folgenden Hinweise, bevor Sie Variablen für Pfade verwenden.</span><span class="sxs-lookup"><span data-stu-id="69096-117">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="69096-120">Stellen Sie sicher, dass die Pfade absolut sind, um Fehler zu vermeiden, die durch Anwendungen verursacht werden, die zu verschiedenen Zeitpunkten an verschiedenen Standorten starten.</span><span class="sxs-lookup"><span data-stu-id="69096-120">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="69096-121">Jede Variable kann in einem Pfad nur einmal vorkommen.</span><span class="sxs-lookup"><span data-stu-id="69096-121">Every variable can occur only once in a path.</span></span> <span data-ttu-id="69096-122">In den meisten Fällen ist dies die einzige sinnvolle Möglichkeit, Variablen zu verwenden, da sie in absolute Pfade aufgelöst werden.</span><span class="sxs-lookup"><span data-stu-id="69096-122">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="69096-123">In fast allen Richtlinien wird der Pfad erstellt, wenn er nicht vorhanden ist (sofern unter den vorhandenen Umständen möglich).</span><span class="sxs-lookup"><span data-stu-id="69096-123">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
- <span data-ttu-id="69096-124">Die Verwendung von Netzwerkspeicherorten für einige Richtlinien kann zu unerwarteten Ergebnissen führen, da sich die verschiedenen Versionen bzw. Kanäle von Microsoft Edge in Bezug auf die Behandlung der Ordnerstruktur unterscheiden.</span><span class="sxs-lookup"><span data-stu-id="69096-124">Using network locations for some policies can lead to unexpected results due to differences in how different versions/channels of Microsoft Edge handle the folder structure.</span></span>
